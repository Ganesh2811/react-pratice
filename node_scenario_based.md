1. Your Node.js API is getting slow when multiple users hit it at the same time. How will you handle and optimize it?
Use logs / monitoring tools, Optimize Code, caching, scaling, DB optimization, Async handling, Handle External APIs, Scaling the Application, Async & Queue Handling.

2. Your API is calling a third-party service, but sometimes that service is slow or fails.
How will you handle this in production?
Error Handling, Timeout Handling, Retry Mechanism, Fallback Response, Caching, Circuit Breaker Pattern, Async / Queue Handling (Rabbit Mq), Logging & Monitoring

3. Your database is getting overloaded due to too many requests. How will you handle it ?
Optimize Database First, Caching, Read vs Write Separation, Rate Limiting, Queue System, Scaling Backend.

4. Your Node.js server is crashing frequently in production. How will you debug and fix it?
check pm2 logs, handle uncaught errors, check memory issue, check ram, cpu and memory, check node version, debug issue, add monitoring logs (winston logger).

5. Your API response time is increasing day by day. How will you investigate and fix it?
analyse logs, measure the time, optimize data query, implementing caching, check extrenal dependencies and ensure efficient code and proper scaling.
